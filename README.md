# Toduo

Приватный портал задач и живой чат для команды из двух человек — в стиле Notion.
Статический сайт (без сборки и без установки), размещается на **GitHub Pages**;
данные, вход, чат и файлы работают через **Supabase**.

**Живой сайт:** `https://gondon2004442.github.io/toduo/` *(после включения Pages)*

## Возможности

- 🔐 Вход по email и паролю
- 💬 Живой чат между участниками (в реальном времени)
- ✓ Доска задач: этапы **ТЗ → В работе → Готово**
- 📎 Вложения к задаче по этапам: **ТЗ/материалы → процесс → результат**, файлы и ссылки
- Общие данные — оба видят одни и те же задачи и переписку

## Технологии

- Чистый HTML / CSS / JavaScript (ES-модули), без сборки
- [Supabase](https://supabase.com): Auth + Postgres + Realtime + Storage
- Хостинг: GitHub Pages

## Настройка (один раз)

### 1. Создайте проект Supabase
1. Зайдите на <https://supabase.com>, войдите, **New project** (бесплатного тарифа хватает).
2. Дождитесь, пока проект развернётся.

### 2. Создайте таблицы и хранилище
1. В проекте: **SQL Editor → New query**.
2. Откройте файл [`supabase/schema.sql`](supabase/schema.sql), скопируйте всё, вставьте и нажмите **Run**.
   Скрипт создаст таблицы, политики доступа, realtime и бакет `attachments`.

### 3. Упростите вход (по желанию, рекомендуется)
- **Authentication → Providers → Email** → выключите **Confirm email**, чтобы
  можно было входить сразу без подтверждения почты.

### 4. Возьмите ключи
- **Project Settings → API**: скопируйте **Project URL** и **anon public key**.

### 5. Подключите сайт к Supabase
Два способа — любой:
- **Проще:** откройте сайт — на экране настройки вставьте URL и anon-ключ
  (сохранятся в браузере, каждый участник вводит один раз).
- **Или в коде:** впишите значения в [`js/config.js`](js/config.js) и закоммитьте.

## Публикация на GitHub Pages

1. Смёржите изменения в ветку `main` (через Pull Request).
2. В репозитории: **Settings → Pages**.
3. **Source:** *Deploy from a branch* → **Branch:** `main` → **Folder:** `/ (root)` → **Save**.
4. Через минуту сайт будет доступен по адресу
   `https://gondon2004442.github.io/toduo/`.

Сборка не нужна — это статические файлы.

---

## Локальный запуск (для проверки)

Установка не требуется. Любой статический сервер, например:

```bash
python3 -m http.server 8000
# открыть http://localhost:8000
```

> Просто открыть `index.html` двойным кликом (`file://`) не получится —
> ES-модули требуют http-сервер.

## Проверка вдвоём

1. Оба откройте сайт и зарегистрируйтесь (каждый — свой email).
2. Напишите сообщение из одного браузера — во втором оно появится сразу.
3. Создайте задачу, прикрепите файл в «ТЗ» и ссылку в «Результат»,
   переключайте статусы — второй участник видит обновления вживую.

## Структура

```
index.html            — страница-оболочка
styles.css            — оформление
js/config.js          — ключи Supabase (или вводятся в браузере)
js/supabase.js        — инициализация клиента
js/auth.js            — вход / регистрация / профиль
js/chat.js            — живой чат
js/tasks.js           — доска задач и вложения
js/app.js             — маршрутизация и оболочка
js/util.js            — вспомогательные функции
supabase/schema.sql   — схема БД, политики, realtime, storage
```
